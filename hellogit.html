Git использует в большинстве компанийб где над проектом работает хотя бы два разработчика:
  1. Новый человек приходнт в компанию н клонирует репозиторий проекта на ПК.
  2. Новый человек приходнт в компанию н клонирует репозиторий проекта на ПК.
  3. Получает задачу, создаёт новую ветку н пишет код.
  4. Когда всё готово - отправляет запрос на добавление кода в master-BeTkY.
  5. Когда всё готово - отправляет запрос на добавление кода в master-BeTkY.
  6. Другие разработчики смотрят код, оставляют комментарии и указывают на ошибки.
  7. Новичок дорабатывает код. обновляет masteT-BeTkY н переходит к следующей задаче.
